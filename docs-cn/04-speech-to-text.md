# 语音转文本学习笔记

## 概览

语音转文本（STT）用于把音频内容转换为文字，常见场景：

- 会议转录
- 语音笔记索引
- 字幕生成
- 通话分析
- 音频档案检索

## 常见模型

语音转文本模型家族示例：

- `gpt-4o-mini-transcribe`
- `gpt-4o-transcribe`
- `gpt-4o-transcribe-diarize`
- `whisper-1`

## 核心模式

### 1. 基础转写

上传音频文件并返回文本。

### 2. 说话人分离（Diarization）

多说话人录音可通过分离能力区分发言者并返回片段元数据。

### 3. 翻译

部分流程可在转写同时将语音内容翻译为英文。

### 4. 提示词

提供领域术语、缩写、话题背景等上下文可提升识别准确率。

## 实战建议

- 文件需控制在大小限制内，超限时先切片。
- 尽量避免在句子中间切分音频。
- 用提示词提升生僻词识别率。
- 需要字幕、剪辑点或词级对齐时启用时间戳。

## Azure Speech to Text 深入（对齐 AI-901）

### 核心模式

- 实时转写：适用于直播交互与辅助场景。
- 批量转写：适用于离线录音处理。
- 说话人分离：适用于多人会议/访谈。
- 自定义语音：适用于领域词汇与声学优化。

### 工程考虑

- 实时模式优先低延迟；批量模式通常优先吞吐。
- 准确率受噪声、麦克风质量、口音与领域术语影响。
- 切片策略应尽量保留句边界，减少上下文丢失。
- 可用置信度与分段元数据将低置信片段路由到人工复核。

### 常见失败点

- 领域词不在词汇覆盖内。
- 说话重叠降低分离质量。
- 过载或削波造成不可逆识别损失。

缓解措施：

- 尽可能加入领域提示或自定义语音训练。
- 做音频预处理（降噪、归一化、声道分离）。
- 对关键转写加入后编辑与纠错流程。

## Azure Speech Translation 深入

语音翻译不只是“STT + 文本翻译”的简单拼接，它面向实时语音到文本/语音的多语言沟通。

### 关键能力

- 语音到文本翻译
- 语音到语音翻译
- 多语言翻译（包括语言切换场景）
- 实时口译模式
- 多目标语言输出

### 成本提示

- 多目标输出会明显增加成本，因为每增加一种目标语言都会增加翻译负载。
- 长会话需同时预算转写与翻译两部分成本。

### 服务选型建议

| 场景 | 推荐基线 |
|---|---|
| 单语会议纪要 | Speech to Text |
| 实时跨语种对话 | Speech Translation |
| 多语言通话后分析 | STT + 批量翻译流水线 |

### AI-901 备考视角

- 能区分“转写”与“翻译”工作负载。
- 理解何时需要自定义语音来提升质量。
- 理解实时与批处理的权衡逻辑。

## 最小 Python 模式

```python
from openai import OpenAI

client = OpenAI()

with open("speech.mp3", "rb") as audio_file:
    transcription = client.audio.transcriptions.create(
        model="gpt-4o-transcribe",
        file=audio_file,
    )

print(transcription.text)
```
