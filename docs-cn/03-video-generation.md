# 视频生成学习笔记

## 概览

OpenAI 视频生成通常是异步工作流。你不会立刻拿到最终视频，而是：

1. 发起渲染任务。
2. 轮询任务状态或等待 webhook。
3. 任务完成后下载最终视频资产。

## 主要概念

### 1. 面向“运动”的提示词

高质量视频提示通常包含：

- 镜头类型
- 主体
- 动作
- 场景
- 光线
- 运镜方式
- 风格/氛围

示例：

- `Wide shot of a paper boat floating across a rain puddle at sunset, soft golden reflections, camera slowly tracks left.`

### 2. 异步流程

典型流程：

- 创建视频任务
- 获得 `video_id`
- 查询状态（如 `queued`、`in_progress`、`completed`、`failed`）
- 完成后下载 MP4

### 3. 引导输入

根据 API 与模型可用性，可能支持：

- 使用参考图作为首帧
- 复用非人角色资产保持一致性
- 延展已有片段
- 用聚焦提示编辑已有视频

## 实战建议

- 迭代阶段优先短片段。
- 调试提示时先用低分辨率。
- 修改要窄而具体。
- 生产系统优先 webhook，避免激进轮询。

## 限制与可用性

相比文本与图像 API，视频生成通常有更严格的安全护栏与可用性限制。落地前请确认：

- 账号已开通目标模型权限。
- 模型未被弃用。
- 支持的内容与输入类型满足你的场景。

## Azure Bot Service 深入（对齐 AI-901）

视频体验常通过对话界面交付。当你的 AI 流程需要对话管理、渠道接入与机器人托管时，Azure Bot Service 很关键。

### 需要理解的点

- Bot Framework SDK：用 C# 或 JavaScript 进行代码优先开发。
- Copilot Studio：低代码/无代码创建 Agent，并可扩展。
- Composer：在 Bot Framework 模式上做可视化流程编排。

### 在视频工作流中的作用

- Bot 可收集用户意图并触发视频生成。
- Bot 可回传异步进度与最终资源。
- 渠道集成可覆盖 Web Chat、Teams 或自定义前端。

### 关键架构模式

1. 用户在 bot 渠道提出生成请求。
2. Bot 校验提示与策略约束。
3. 后端排队执行视频生成。
4. Bot 推送进度事件并返回最终下载链接。

## 视频与多模态流水线中的 Azure Machine Learning

基础 API 驱动视频生成并不一定需要 AML；但当你要做排序、质量预测、审核分类时，AML 会更重要。

### 媒体工作流中常见 AML 增强

- 训练生成片段质量打分模型。
- 训练领域定制安全分类器。
- 用 MLOps 控制实验追踪与打分端点部署。

### 决策规则

- 先用托管模型 API 完成生成。
- 当自定义后处理模型变成可靠性/合规刚需时再引入 AML。

## Agent 媒体系统中的 LUIS 与意图路由

尽管 LUIS 已属于偏历史/归档组件，但“意图 + 实体”设计仍是 Bot 与 Agent 路由基础。

可复用思路：

- 将“意图识别（用户要做什么）”与“参数抽取（有哪些约束）”分离。
- 按意图路由到不同媒体工具：图像生成、视频生成、翻译、转写等。

## 建议练习

1. 为同一场景写三种电影风格提示。
2. 设计一个视频生成后端轮询流程。
3. 比较何时用参考图、何时用视频延展。
4. 草拟“渲染完成”webhook 处理器。
