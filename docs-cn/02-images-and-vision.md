# 图像与视觉学习笔记

## 核心使用场景

OpenAI 图像工作流通常分成两类：

- 使用视觉模型理解图像
- 使用图像生成工具创建或编辑图像

## 视觉基础

模型可以查看一张或多张图片，并回答关于内容的问题。常见用途：

- 图像描述（Captioning）
- 类 OCR 文本抽取
- UI 理解
- 视觉质量检查
- 商品目录信息补全

## 图像输入方式

可通过以下方式传入图像：

- 公网图片 URL
- Base64 Data URL
- 通过 Files API 上传后使用 file ID

## 细节等级

视觉请求通常支持 `low`、`high`、`original`、`auto` 等细节等级。细节越高，保真度通常越好，但成本也可能更高。

## 图像生成基础

生成图像时，建议描述主体、风格、场景、光线、构图与约束。更好的提示通常包含：

- 主体
- 环境
- 镜头构图
- 光线
- 配色
- 视觉风格

## 实战建议

- 先用短提示起步，仅在输出跑偏处增加精确信息。
- 明确写出“不希望出现”的元素。
- 先用于创意探索，再通过迭代提示精修。
- 注意图像输入也会计入 token 或图像成本。

## Azure Computer Vision 深入（对齐 AI-901）

Azure Computer Vision 侧重从图像中抽取结构化信号；生成式模型侧重创建/变换内容。考试场景常要求你在两者之间做选择。

### Computer Vision 核心能力分组

- 图像分析：标签、描述、场景级理解。
- OCR：从图像中提取印刷文本。
- 领域能力：面向场景的抽取与索引流程。

### 企业常见场景

- 用图像元数据补全商品目录。
- 票据、标识文本抽取并进入下游处理。
- 内容审核与视觉合规检查。
- 通过图像描述提升无障碍体验。

### 服务选型建议

| 需求 | 更适合 |
|---|---|
| 需要确定性的图像元数据与 OCR 输出 | Azure Computer Vision |
| 需要按文本创作图片 | 图像生成模型 |
| 需要结合业务上下文做复杂图文推理 | 多模态 LLM 工作流 |

### 设计与可靠性提示

- OCR 质量依赖分辨率、倾斜、光照与语言覆盖。
- 图像描述是摘要，不等于完整目标清单。
- 生产管线建议同时保存原始输出与标准化字段，便于后续重处理。
- 在高风险场景中，视觉抽取应配合人工复核阈值。

### AI-901 备考视角

- 明确区分“感知/抽取”与“生成”。
- 预期会遇到 OCR + NLP + 搜索/报表组合题。
- 能基于输出类型、延迟与可解释性说明选型理由。

## 最小视觉示例

```python
from openai import OpenAI

client = OpenAI()

response = client.responses.create(
    model="gpt-4.1-mini",
    input=[
        {
            "role": "user",
            "content": [
                {"type": "input_text", "text": "Describe the key objects in this image."},
                {"type": "input_image", "image_url": "https://example.com/sample.jpg"},
            ],
        }
    ],
)

print(response.output_text)
```
